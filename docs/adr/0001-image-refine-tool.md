# ADR 0001: 基于 AGNES i2i 的图片精修工具

**日期**: 2026-07-18

## 状态

已采纳

## 背景

AGNES API 提供了图生图 (i2i) 能力，可以基于输入图片 + 文本指令生成新图片。我们希望基于此能力构建一个图片精修工具，覆盖画质增强、局部修改、风格迁移三类操作。

## 决策

### 1. 全部依赖 AGNES i2i

所有精修操作（包括去噪、锐化等传统上更适合用 OpenCV 处理的操作）统一走 AGNES i2i API，不引入传统图像处理库。

**原因**: 保持技术栈单一，降低维护成本。AGNES i2i 的生成能力可以覆盖所有操作类型，虽然画质增强类操作不如传统方法精确，但胜在统一接口。

### 2. 两种模式

- **工具模式**: CLI 脚本，一次输入一张图 + 操作指令，直接输出结果
- **工作流模式**: 通过 AI 对话驱动，支持多步流水线

### 3. 配置管理

预设风格存储在 `~/.config/agnes/refine-presets.yaml`，与 API key 同目录，方便管理。

### 4. 代码位置

所有精修相关脚本放在 `~/.agents/skills/agnes-api/scripts/` 下，与现有 AGNES 脚本同目录。

### 5. 输入输出

- 输入支持本地文件路径和 URL
- 输出另存为新文件，不覆盖原图
- 输出目录：`~/agent/media/images/`

### 6. 引擎

全部操作基于 AGNES i2i API（`agnes-image-2.1-flash`），通过文本指令驱动，不引入传统图像处理库。

## 备选方案

### 混合引擎

引入 OpenCV/PIL 处理画质增强类操作（去噪、锐化、色彩校正），AGNES i2i 只处理风格迁移和局部修改。

**被拒绝原因**: 用户明确要求完全依赖 AGNES i2i，保持技术栈统一。

### 独立 skill

将精修工具拆分为独立 skill `image-refine`。

**被拒绝原因**: 与 AGNES API 强耦合，放在同一 skill 下更合理，减少维护成本。
