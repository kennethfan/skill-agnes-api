# AGNES API — Domain Glossary

## 核心概念

| 术语 | 定义 |
|------|------|
| **精修 (Refine)** | 对图片进行画质增强、局部修改或风格迁移的处理过程 |
| **工具模式** | 通过 CLI 命令直接调用精修操作，一次处理一张图 |
| **操作 (Operation)** | 一次精修动作，如去噪、调亮、换背景 |
| **流水线 (Pipeline)** | 多个操作按顺序组合执行 |
| **管线 (Pipeline)** | 将视频制作拆分为三阶段（场景图生成 → 素材准备 → 视频合成）的自动化流程 |
| **Checkpoint** | 记录管线各阶段的完成状态的 JSON 文件，用于中断后断点续传 |

## 精修操作

| 术语 | 定义 |
|------|------|
| **画质增强** | 改善图片视觉质量的操作集合，包括去噪、超分、调亮、色彩校正、锐化 |
| **局部修改** | 对图片局部区域进行编辑，包括换背景、去除物体、替换颜色、添加元素 |
| **风格迁移** | 将图片转换为特定艺术风格，包括预设风格和自定义风格 |

## 配置

| 术语 | 定义 |
|------|------|
| **预设风格** | 预定义的艺术风格模板，存储在 YAML 配置文件中，可通过文本描述驱动 i2i 生成 |
| **路径配置** | `~/.config/agnes/paths.yaml` — 所有输出路径的唯一信源，脚本通过 `utils.py` 读取 |

## 文件组织

| 术语 | 定义 |
|------|------|
| **Asset** | Pipeline 的输入素材（角色参考图、背景图），不可再生，保留 |
| **Intermediate** | Pipeline 生成的临时文件（场景图、逐格、气泡、视频帧、音频段），可以安全删除，完成后询问用户 |
| **Deliverable** | Pipeline 的最终交付物（拼页 PNG、视频 MP4），保留 |
| **Project** | 文件组织的一级单位。每个项目一个独立目录 `~/agent/media/projects/<name>/`，包含自己的剧本、intermediates 子目录和 deliverable 子目录 |
| **Cleanup Prompt** | Pipeline 完成后询问用户是否删除 intermediates 目录，默认 yes |
| **Default Dir** | `~/agent/media/tmp/` — 无项目归属的零散生成文件兜底目录 |

## 故事视频

| 术语 | 定义 |
|------|------|
| **故事视频 (Story Video)** | 面向 3-6 岁儿童的动画故事视频，AI 生成场景图 + TTS 旁白 + 纯文字字幕，输出 mp4 |
| **场景 (Scene)** | 故事视频中的一段情节单元，对应一个画面 + 一段旁白文字 |
| **分镜 (Storyboard)** | 将故事文本拆解为多个场景的过程，由 AI agent 自动完成 |
| **角色锚定 (Character Anchoring)** | 首张场景使用 t2i 生成，后续使用 i2i 以首张为参考保持角色外观一致的策略 |
| **标题卡 (Title Card)** | 视频首屏的故事名封面，用 Pillow 在场景图上叠加标题文字 |
| **故事脚本 (Script)** | 描述故事视频内容的 YAML 文件，包含标题、风格、语音和场景列表 |
| **对话 (Dialogue)** | 场景中的一个角色台词片段，包含角色名、文字和语音标识 |
| **角色语音映射** | AI 根据角色类型自动分配 edge-tts 中文语音，可在 YAML 中手工覆盖 |
